Integração micronaut-langchain4j: DI de compilação (e a versão real)

Tudo que você aprendeu da API standalone agora entra no Micronaut 5 via injeção de dependências em tempo de compilação. E a primeira lição é uma verdade verificada na fonte: a versão que o seu stack realmente recebe.

Por que isto importa para a sua missão — em produção, o assistente do banco não é montado "na mão": o Micronaut fia o modelo Bedrock, as tools, a memória e o RAG por DI de compilação (sem reflexão em runtime, ideal para GraalVM). Mas você precisa saber qual versão do LangChain4j o módulo resolve — senão ensina uma API que o seu build não tem.

1 · A verdade da versão (manchete verificada)

O micronaut-langchain4j 2.0.1 NÃO embarca 1.16.2. O catálogo fixa managed-langchain4j = "1.15.1", managed-langchain4j-community = "1.15.0-beta25", micronaut = "5.0.1"; o BOM importado é o langchain4j-bom 1.15.1.

O que fazer (Foundation First — saber a versão real é competência):

  • (a) usar a superfície como 1.15.1 — as APIs centrais (@Tool, @P, AiServices, BedrockChatModel) são estáveis entre 1.15 e 1.16; ou
  • (b) sobrescrever o langchain4j-bom para 1.16.2 e re-verificar os deltas — recursos que ensinamos em 1.16.2 (capability de JSON schema da Lição 10, framework de guardrails @Experimental da 11, BedrockGuardrailConfiguration, promptCaching da 8) podem diferir ou não existir em 1.15.1.

2 · O que o módulo faz por você (DI de compilação)

RecursoComo (verificado @ v2.0.1)
Beans de modelo Bedrock@Lang4jConfig em BedrockModule → processador gera @ConfigurationProperties/@Factory → BedrockChatModel (Converse)
Declarar AI service@AiService numa interface — vira bean com DI de compilação

3 · Configurar o Bedrock (armadilhas verificadas)

  • Prefixo código-verdade: langchain4j.bedrock (.model, .region). ⚠️ A doc oficial mostra langchain4j.bedrock-llama — obsoleto/incorreto em 2.0.1 (que não declara modelo Llama). Copiar a doc literalmente dá um prefixo que não funciona.
  • Autenticação = credenciais AWS, nunca api-key. credentialsProvider é injected=true, required=true → registre um bean AwsCredentialsProvider (ex.: micronaut-aws SDK v2). O api-key do BedrockTest é só gatilho de teste.
  • Model default = claude-3-haiku-20240307 (alias). Em produção, defina um model id / inference profile explícito via langchain4j.bedrock.model.

4 · @AiService, @Tool e a dívida do @P (resolvida)

@AiService (io.micronaut.langchain4j.annotation): value()/named() escolhem o bean de modelo; tools() lista as classes de tools; customizer() ajusta o builder.

Tools só são fiadas se listadas em @AiService(tools={...}) — getAllTools() não é aplicado automaticamente, e os métodos @Tool precisam estar em beans @Singleton. Esquecer um dos dois = o modelo nunca enxerga a tool.

★ Dívida de verificação resolvida — em 1.15.1, @P tem name(). O plugin Gradle do Micronaut adiciona -parameters por padrão (PR #153) → nomes simples de parâmetro já funcionam. Mas um build Maven puro sem <parameters>true</parameters> gera arg0/arg1 (confunde o modelo) → aí @P(name=...) é obrigatório.

5 · O assistente do banco, fiado por DI

# application.yml — SOURCE-OF-TRUTH prefix for 2.0.1
langchain4j:
  bedrock:
    model: anthropic.claude-3-5-sonnet-20240620-v1:0   # explicit in prod (not the haiku alias)
    region: us-east-1
# authentication = AWS credentials → register an AwsCredentialsProvider (micronaut-aws SDK v2)
@Factory
class AwsConfig {                                  // bean REQUIRED for Bedrock to authenticate
    @Singleton AwsCredentialsProvider creds() { return DefaultCredentialsProvider.create(); }
}

@Singleton                                         // @Tool methods MUST live in a @Singleton
class BankingTools {
    @Tool("Consulta o saldo de uma conta (nacional ou global)")
    String saldo(@P("numero da conta") String accountId, @P("escopo: NACIONAL|GLOBAL") String scope) { /* ... */ }

    @Tool("Transfere valor; exige idempotencyKey para evitar dupla execução")
    String transferir(@P("conta origem") String from, @P("conta destino") String to,
                      @P("valor em centavos") long amountCents,   // primitive: required IS validated in 1.x
                      @P("chave de idempotência") String idempotencyKey) {
        // confirmation + idempotency live HERE (the framework only routes the call)
    }
}

@AiService(tools = { BankingTools.class })         // tools ONLY wire in if listed here
interface BankingAssistant {
    @SystemMessage("Você é um assistente bancário. Nunca efetive transferências sem confirmação.")
    String chat(String userMessage);
}
// inject BankingAssistant into any @Controller/@Singleton

6 · Pratique (recuperação ativa)

Quiz — recupere de memória, sem reler

1. Qual versão do LangChain4j o micronaut-langchain4j 2.0.1 resolve por padrão?

O catálogo de versões da tag v2.0.1 fixa managed-langchain4j = 1.15.1 e importa o langchain4j-bom 1.15.1. Para ter 1.16.2, sobrescreva o BOM e re-verifique deltas.

2. Uma classe @Tool não aparece para o modelo. Causa mais provável?

O AiServiceFactory só chama builder.tools() para classes listadas em @AiService(tools={...}); e o ToolRegistry só registra @Tool em beans @Singleton.

3. Como o BedrockChatModel autentica sob micronaut-langchain4j?

credentialsProvider é injected=true, required=true: registre um AwsCredentialsProvider (ex.: micronaut-aws). O api-key do BedrockTest é só gatilho de teste.
