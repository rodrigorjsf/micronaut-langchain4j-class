Arquitetura de referência: o backend agêntico do banco, ponta a ponta

Esta lição não traz API nova — ela monta tudo o que a trilha aterrou numa arquitetura escalável, segura e performática, em dois níveis: macro (componentes e integrações) e micro (o fluxo seguro de uma requisição, com a decisão e o impacto em cada gate).

Por que isto importa para a sua missão — é aqui que as peças viram sistema: o loop agêntico, as tools determinísticas, os guardrails, as skills, o RAG, a observabilidade e a escala — todos sobre Claude Sonnet via AWS Bedrock. Cada componente abaixo aponta para a lição que o verificou; nada de API solta de memória.

1 · Macro — componentes e integrações

A requisição entra pela borda, desce pela camada agêntica e pela camada determinística de tools, e se integra ao Bedrock, ao core bancário, ao RAG e à observabilidade. O cérebro (o loop) é a parte não-confiável; as travas reais ficam fora dele.

2 · Micro — o fluxo seguro de uma requisição

Defesa em profundidade: cada gate é uma decisão, e o caminho de falha tem um impacto mapeado ao OWASP LLM Top 10 (Lição 15). A regra-mestra: o modelo propõe; a sua camada determinística autoriza.

★ Insight — a segurança é arquitetural, não promptual. Repare que todo gate do diagrama micro vive na camada determinística (azul/dourado), nunca no texto que o modelo lê. Identidade, autorização, validação, idempotência: tudo fora do alcance do prompt injection. O loop agêntico é poderoso porque é cercado — não apesar disso.

4 · Honestidade de plataforma: o que o Bedrock dá e o que é DIY

Uma arquitetura honesta nomeia o que você constrói:

  • Loop agêntico: você o monta com AiServices — o banco não usa managed agents; o Bedrock é o caminho de invocação do modelo, não o orquestrador (Lição 19).
  • Resiliência: não há FallbackChatModel nem rate limiter prontos — retry/throttling/fallback são seus (Lição 18).
  • Guardrails: os do LangChain4j são @Experimental; em produção, combine com Bedrock Guardrails (Lição 11).
  • Avaliação: sem camada nativa em Java — o juiz é um prompt + modelo forte que você escreve (Lição 21).

5 · Pratique (recuperação ativa)

Quiz — recupere de memória, sem reler

1. Onde vive a autorização de uma ação que move dinheiro, na arquitetura de referência?

O modelo propõe; quem autoriza, valida e executa é a camada determinística — nunca o texto que o modelo lê.

2. No fluxo micro, qual o impacto OWASP do gate "Ação autorizada?"

O gate de autorização determinística é a defesa contra excessive agency (LLM06): nenhuma ação roda só porque o modelo a sugeriu.

3. Por que o banco NÃO usa managed agents do Bedrock como orquestrador?

O banco monta o próprio loop com AiServices (LangChain4j); o Bedrock é o caminho de invocação do modelo, não o orquestrador.
